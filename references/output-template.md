# 输出模板

生成的文件名格式：`{project-name}-spec.md`
保存位置：用户当前工作目录

---

以下是模板内容，`{变量}` 为占位符，根据问答结果填充。

---

|------|------|

---

## 模板使用规则

1. 根据项目类型裁剪 Section：
   - 纯前端 → 省略"数据模型"
   - CLI 工具 → 省略"系统架构"中的前端部分和"数据模型"
   - 库/SDK → 省略"数据模型"，"目录结构"调整为库的标准结构

2. 技术栈表格只保留用户选择了的行，没选的不要空着

3. 实施计划的具体任务根据用户的核心功能列表填充

4. 目录结构根据用户选的框架生成真实的结构（如 Next.js 的 app/ 目录，FastAPI 的 routers/）
